# Claude Design 依頼書 — ふるさとピザ帳 / Slice 8 機材ガイド

> 「ふるさとピザ帳」(技術名: MakeLocalPizzaRecipeAgent) の **Slice 8 (機材ガイド + 機材プロファイル)**
> で新規追加する画面・UI 要素のデザインを依頼します。
> Slice 7 (フィードバック) まで本番稼働中
> (https://mlpr-web-343527548585.asia-northeast1.run.app)、Slice 8 はこれから実装。

## サービス概要 (前提)

- 地元 / 旬の食材を活かしたピザレシピを提案する Web アプリ
- Quick Tap UX: 地元 → 食材 → 候補3案 → 詳細 → ハート保存 → 作ってみた振り返り
- 戦略軸: 王道 (exploit, 朱) / 一歩外す (tune, 黄柏) / 大冒険 (explore, 鴬)
- デザイントーン: **和モダン** (washi テクスチャ・明朝 + ゴシック + モノ・墨と朱を主軸)
- ターゲットユーザー: ピザパーティを主催するホスト (3-8 名規模)
- サービス名: **「ふるさとピザ帳」**

## Slice 8 で何をしたいか (1 行)

**「家庭オーブンでは到達できない 400℃ の焼成体験を、初心者ホストにも届ける」**ため、
ENRO 電気ピザ窯を推奨機材として明示し、機材プロファイル切替と独立ガイドページを追加する。

## 背景 (デザイン判断の根拠)

- 開発者自身が ENRO (電気ピザ窯) を使って「家庭オーブンでは出せない店レベルのナポリ寄り
  仕上がり」に到達した実体験が本アプリの物理的裏付け
- ENRO の独自価値は **(1) 400℃ 級の高温焼成が電源運用で屋内常用できる、
  (2) ポータブル性 — レンタルキッチンや友人宅に持ち込み「その場で焼く」体験を可能にする、
  (3) 個人購入できる価格帯** の 3 点
- ただし全アプリを「ENRO 専用」にしたくはない。**ENRO 推奨 + 家庭用オーブン補足**
  という二段構えで、間口は維持しつつ ENRO を主役に据える
- 楽天アフィリエイトを採用するが、開発者は ENRO の **1 ファン**としての立場
  (メーカー提携・案件ではない) を明示する

## 依頼内容 — Slice 8 で新規 / 改修する 3 + 2 ブロック

### 1. `/equipment` 機材ガイドページ (★ 最優先・新規ルート ★)

**位置付け**: 認証不要・静的ページ。本 Slice の主役。
**Phone frame** (393 × 852) ではなく、**通常の縦長 LP** 想定 (スクロール前提)。
ただしトーンは既存アプリと完全に統一(washi 背景・明朝 / ゴシック / モノの使い分け)。

#### 1-1. 構成セクション (上から下へ)

1. **Hero**
   - 大きめキャッチ「**400℃の窯が、家で焼ける。**」(明朝、大)
   - サブコピー「**ふるさとピザ帳が前提にしているのは、ENRO の電気ピザ窯です。**」(ゴシック)
   - 機材画像 (placeholder、後で本物画像差し替え) — 1 枚大きく
   - 一次 CTA「ENRO を見る (楽天)」(ShuButton、external link icon)
   - 二次 CTA「アプリで食材を選ぶ」(ghost / matcha)

2. **なぜ ENRO か — 開発者の声 (一人称ブロック)**
   - eyebrow: 「開発者から」
   - 短い一人称テキスト (200〜300 字):
     「仙台で地元食材を活かしたピザパーティをやっているうちに、
      家庭オーブンでは越えられない壁があることに気付きました。
      ENRO の電気ピザ窯を導入してからは、縁が膨らみ、中央がしっとりした
      "店レベル" の一枚が家で焼けるようになりました。
      このアプリのレシピは、その体験を前提に最適化されています。」
   - アイコン or イラスト (小、ピザ + 窯)
   - **重要**: 案件感を出さず、ブログのような温度で

3. **3 つの体験変化 (3 カラム or 縦積み 3 ブロック)**
   - カード 1: 🔥 **400℃ の高温焼成**
     - 「90〜120 秒で焼き上げる短時間高温が、ナポリピザ本来の食感を生む」
   - カード 2: 🎒 **ポータブル運用**
     - 「電源さえあればレンタルキッチン / 友人宅 / 屋外イベントに持ち出せる。
        その場で焼きたてを振る舞うのが本来のピザ体験」
   - カード 3: 💰 **個人で買える価格帯**
     - 「業務用石窯と違って、ホスト個人で導入できる価格と設置サイズ」
   - スタイル: kinari 地に sumi 文字、見出しは明朝、本文ゴシック
   - **特に「ポータブル運用」はこのプロダクトのコンセプトと深く繋がるので、強めに**

4. **基本の使い方 (5 ステップ + 比較表)**
   - eyebrow: 「ENRO で焼く基本手順」
   - ステップ:
     1. 予熱 (15〜20 分で 400℃ 到達)
     2. 生地を伸ばす
     3. ソース + トッピング
     4. 投入 (専用ピール推奨)
     5. 90〜120 秒で焼き上がり (途中で 180°回転)
   - **比較表**: 家庭オーブン vs ENRO (温度 / 時間 / 仕上がり) — washi 地の表
   - 公式 YouTube への外部リンク (動画リスト 2〜3 本、各サムネ + タイトル)

5. **家庭用オーブンしかない場合 (補足セクション、明確に控えめなトーン)**
   - eyebrow: 「機材を揃える前に試したい方へ」
   - 「アプリ内の設定で **機材プロファイル**を **家庭用オーブン**に切り替えると、
      レシピが 250〜300℃ / 8〜15 分前提に再生成されます。」
   - 機材プロファイル切替 UI のスクリーンショット風イラスト
   - 「ただし、本アプリの本領は ENRO による高温焼成にあります」と一言添える
   - CTA「アプリで機材プロファイルを変更する」(設定 or ヘッダのプロファイル UI へ)

6. **CTA セクション (フッター手前)**
   - 大きく「ENRO を見る (楽天)」ShuButton
   - 「ふるさとピザ帳を試す (食材を選ぶ)」secondary

7. **フッター注記**
   - 小さいテキストで:
     「※ 本ページのリンクは楽天アフィリエイトを含みます。運営者は ENRO の 1 ユーザとして
      本機材を推奨していますが、メーカーとの提携・PR 案件ではありません。」
   - washi 地に muted gray、12px 想定

#### 1-2. 欲しい状態

- 通常表示 (スクロールしてすべて見える状態)
- Hero だけ単独 (above the fold)
- 比較表のクローズアップ
- フッター注記のクローズアップ (透明性の見せ方を確認したい)

#### 1-3. デザイン上の論点 (提案ほしい)

- **画像枠 (機材写真) のプレースホルダ表現** — 実機材写真は後で差し替える前提
- **比較表のデザイン** — 既存パターンにテーブルが少ない。和紙地のカード分割?
- **「ポータブル運用」をどう視覚的に強調するか** — アイコン or 小イラスト or 写真
- **アフィリエイト注記の位置** — フッター固定 vs ページ末尾。情報設計として透明性を担保したい

### 2. 機材プロファイル切替 UI (新規コンポーネント・全画面で共通)

**目的**: ユーザが「ENRO」と「家庭用オーブン」を切り替えられるようにする。
切替はレシピ生成プロンプトに直結。

#### 2-1. 配置候補 (提案ほしい)

- **案 A**: HeaderRow の Dropdown メニュー内に「🔥 機材: ENRO ▾」項目を追加
  (Slice 7 で導入した Dropdown に 1 項目足す)
- **案 B**: 食材選択画面 (Tap2) のヘッダ近辺に独立した小さなセレクタ
  `🔥 ENRO ▾` を配置 (地元表示 `📍 [地元名] ▾` と並列)
- **案 C**: 設定画面を新設し、そこで切り替える (重め)

**現状の好み**: **案 A + 案 B のハイブリッド**
- 普段はメニュー内 (案 A) で「現在のプロファイル」が見えるように
- 食材選択画面 (Tap2) でだけ案 B の独立セレクタを出す
  (食材選択 = レシピ生成直前なので、確認しやすい位置に出す)

#### 2-2. プロファイル選択モーダル / ボトムシート

セレクタを押したときに何が出るか:

- 2 つのオプション (`enro_450c_90s` / `home_oven_280c_10m`)
- 各オプションに以下が並ぶ:
  - アイコン (🔥 / 🍳)
  - 表示名 (ENRO 電気ピザ窯 / 家庭用オーブン)
  - 推奨温度・時間 (400〜450℃ / 90〜120 秒 vs 250〜300℃ / 8〜15 分)
  - 短い説明
- 一番下に「機材ガイド (/equipment) を見る」リンク
- 確定ボタン

**欲しい状態**: ENRO 選択中 / 家庭用オーブン選択中 / 切替直後のフィードバック (Toast?)

### 3. HeaderRow Dropdown への「機材ガイド」項目追加 (Slice 7 既存改修)

**「🔥 機材ガイド」→ /equipment** リンクを 1 項目追加してほしい。

- 位置: 「📓 振り返り帳」の下、区切り線の前

  (たとえば項目に「🔥 機材ガイド」+ 副題「現在: ENRO」)

## 制約 / 注意事項

- **トークン体系** (`design/pizza-tokens.jsx`) を遵守。新色は追加しない方向で
- **明朝 = 余韻 / ゴシック = 機能 / モノ = メタデータ** の使い分けを継続
- 既存パターン (`Phone` / `BackChip` / `Chip` / `ShuButton` / `Stat` / HeaderRow) を再利用
- `/equipment` は **Phone frame ではなく LP**。ただし内部の素材は既存トークンと合わせる
- iOS frame 想定画面 (機材プロファイル切替モーダル等) は 393 × 852、SAFE_TOP / SAFE_BOT は既存値
- **「ENRO 専用感」が出すぎないこと** — 家庭用オーブン補足の存在が伝わるように
- **アフィリエイト透明性** — `rel="sponsored"` 想定 + ページ末尾に明示注記
- **トーン**: 「PR / 案件」ではなく「個人ユーザの強い推奨」。明朝で語りすぎず、
  ゴシックで淡々と機材の良さを説明するバランス
