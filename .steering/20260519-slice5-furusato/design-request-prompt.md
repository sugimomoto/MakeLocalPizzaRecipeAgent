# Claude Design 向けプロンプト — Slice 5 ふるさと納税セクションの設計依頼

> このファイルをそのままコピペして Claude Design に投げる想定。
> 出力は Figma 互換 / HTML モックアップ / 画像どれでも可。
> 既存の Slice 1〜4 (TOP / /local / /ingredients / /candidates / /recipes / /library / SignInModal / Toast 等)
> はそのままで、**詳細画面 (`/recipes/[id]`) に新セクションを 1 つ追加** + その周辺の状態案を出してほしい。

---

## アプリ概要

**Make Local Pizza Recipe Agent** — 47 都道府県の地元食材と AI を組み合わせて
「今宵の一枚」のピザを 3 案提案し、気に入った 1 つを詳細レシピ + 画像 + 「ピザ帳」保存
までを 1 つのモバイル Web アプリで完結させる、ハッカソン提出作。

- フレームワーク: Next.js 16 (App Router) + React 19 + TypeScript + CSS Modules
- スタイル方針: **「和紙の上の活版印刷」** をモチーフにした大人向けの落ち着いた和モダン
- 想定デバイス: スマホ縦持ち first (max-width 480px 中央寄せ)
- 既存 12 画面 (TOP / /local / /ingredients / /candidates / /recipes / /library + Toast / SignInModal / AvatarButton 等) は同じトーンで完成済 (v0.4.0)
- **Slice 5 ではこの上に「ふるさと納税で取り寄せる」体験を 1 セクション分だけ載せたい**

---

## Slice 5 のゴール (1 行)

詳細レシピ画面で「**このレシピの食材は、ふるさと納税の返礼品としても入手できます**」を
出し、自治体・寄付額・楽天市場へのリンクをユーザに届ける。

**ハッカソンのコンセプト「つくる × まわす × とどける」のうち、エンドユーザに直接届く
「とどける」(食材の取り寄せ) を担う**。

## 依頼: 以下 3 つの状態 + 1 つの共通要素をデザインしてください

### 画面 1 (主役): ふるさと納税セクション (default state)

**役割**

- 詳細画面の「手 順」と「StoryCard」の間に挿入されるセクション
- このレシピで使う食材を、ふるさと納税の返礼品として取り寄せられるよう、楽天市場 (アフィリエイト URL) へ橋渡し
- 食材ごとに 1〜3 件、最大 12 件のカードを縦に並べる

**セクション構成**

| 要素 | 内容 |
|---|---|
| SectionLabel (mono 11px, 字間 0.3em) | `🎁 ふるさと納税で取り寄せる` または 別アイコン提案歓迎 |
| サブコピー (gothic 12px, sumi-soft) | 「このレシピの食材は、ふるさと納税の返礼品としても入手できます。」 |
| カードリスト | 縦並び、各カード 1 item、最大 12 件 |
| フッタ (mono 10px, sumi-muted) | クレジット表記「Powered by 楽天ウェブサービス」(楽天規約 §8 必須) |

**各カードの要素 (Item card)**

| 要素 | 内容 | フォント / サイズ |
|---|---|---|
| サムネ画像 | 楽天 API の medium image (~128×128) を 72px or 80px 角で round 8 corner | — |
| 商品タイトル | 「【ふるさと納税】宮城県松島町 三陸産生牡蠣...」など | mincho 13〜14px / 2 行 ellipsis / sumi |
| 自治体 | 「宮城県松島町」 | gothic 11px / sumi-soft |
| 生産者 (任意) | 「松島漁業協同組合」 | gothic 11px / sumi-muted |
| 寄付額 | 「寄附 12,000 円〜」 | mono 12px / kogane (黄金) または sumi |
| 在庫切れバッジ | 「在庫切れの可能性」(`inStock=false` のとき) | mono 10px / shu-deep |
| クリック挙動 | カード全体 / または「取り寄せる →」 chip で `affiliateUrl` を新タブ openかつ `rel="noopener noreferrer sponsored"` | — |

**質問: カードのトーン**

3 案ぐらいデザインしてほしい:

- **案 A** — 既存 LibraryCard と同じ感じ (kinari 背景 + hairline + 横並び)。一貫性が高い
- **案 B** — もう少し「外部リンク」感を出して右上に外部リンクアイコン or 楽天マークを薄く
- **案 C** — 黄金 (`--mlpr-kogane`) アクセントを利かせて「特別な取り寄せ品」感

**想定レイアウト (案 A 例)**

```

│ 🎁 FURUSATO / ふるさと納税で取り寄せる   ← SectionLabel
│                                             │

│ としても入手できます。                       │
│                                             │

│ │ [img]  【ふるさと納税】宮城県松島町       │ │

│ │        宮城県松島町 / 松島漁業 協同組合  │ │
│ │        寄附 12,000 円〜            [→]   │ │
```

### 画面 2: 空状態 (`items.length === 0`)

**役割**

- 詳細レシピは出たが、その食材のふるさと納税返礼品がキャッシュに無い場合
- 楽天 API のキャッシュは TTL 7 日で refresh script が走っていない時間帯 / 該当食材が refresh 未対応の場合に発生

**選択肢: そもそも「セクション自体を出さない」or 「セクションは出すが空であることを示す」**

- 案 X: items=0 ならセクション全体を **非表示** (シンプル、UI ノイズなし)
- 案 Y: 「該当の食材はふるさと納税未対応です」と小さく注釈 (情報伝達優先)

→ どちらが詳細画面の流れとして気持ちよいか判断してほしい (個人的には案 X 推し、ただし refresh 未走時には Y で「準備中」表現もアリ)

---

### 画面 3: ロード中 (`state: 'loading'`)

**役割**

- Firestore onSnapshot がまだ最初のスナップショットを返していない 100ms 程度の間

**選択肢:**

- 案 P: セクションごと skeleton (グレーバー + shimmer)
- 案 Q: 何も出さない (loading は短時間なのでチカチカ防止)
- 案 R: SectionLabel だけ出して中身だけ skeleton

→ こちらも判断してほしい (loading が短いなら案 Q が落ち着く)

---

### 画面 4: クレジット表記 (Powered by 楽天ウェブサービス) の表現

**役割**

- 楽天規約 §8 で「楽天ウェブサービスを利用したことが分かる表記」が必須
- 過剰に主張せず、しかし規約を満たすデザイン

**要素**

- 文言: 「Powered by 楽天ウェブサービス」(or 「楽天ウェブサービスのデータを利用しています」)
- 配置: ふるさと納税セクションのフッタ (sub-section)
- フォント: mono 10px / sumi-muted / letter-spacing 0.2em
- 任意で楽天のロゴアイコンを薄く (公式 SVG を使う想定。ロゴ無しでも OK)

**質問:**
- ふるさと納税セクション専用のフッタにするか、画面全体の bottom (stickyCtas の下) にするか
- 公式ロゴを使うかどうか (使うと「お洒落」だが許諾範囲確認が必要)

## 制約

- **モバイル幅 480px 中央寄せ** が主
- 既存トークン (上記カラー / タイポ) を尊重 (新色は最小限)
- 朱色 CTA は画面 1 つにつき 1 つを基本 — **詳細画面では既に「作ってみる →」が朱なので、ふるさと納税カードのクリック誘導は朱を避けて kogane / sumi-soft / ghost 系で**
- 既存の「戦略印」「sticky CTA」「StoryCard」の余白・トーンに馴染ませる
- アフィリエイトリンクであることを暗黙的に伝える (sponsored / 楽天ロゴ等)
- 「ふるさと納税」「楽天」の文字は必ず明示 (規約上)
- 在庫切れの可能性を分かるように出す (`inStock=false`)

## 既存と新規の干渉ポイント

- **既存の「作ってみる →」** sticky CTA (画面下に朱、固定) は変わらない
- **既存の StoryCard** は和紙のシートで「物語」を出すパネル。ふるさと納税セクションはその直前に入る
- **既存の SectionLabel**「食 材」「手 順」と並べる場合、命名は「食 材」「手 順」「取り寄せ」or「贈り物」「お取り寄せ」など 漢字 2 字 + スペースの和テイストも提案歓迎

## 成果物

各状態について:
1. モバイル 480px 幅のフレーム
2. default / loading / empty の 3 状態
3. カードのトーン提案 (案 A / B / C) を比較できる形で
4. 採用色・字サイズ・余白の数値が分かるアノテーション付き
5. クレジット表記の配置例
