## 1. 開催概要

- **イベント名**: DevOps × AI Agent Hackathon
- **主催**: ファインディ株式会社（Findy Tools）
- **メインスポンサー（Diamond）**: グーグル・クラウド・ジャパン合同会社
- **その他スポンサー**: elasticsearch K.K. ほか
- **コンセプト**: 「**つくる、まわす、とどける**」
  - **つくる** — Google Cloud の AI を中核に、実務で役立つ独創的な AI エージェントを設計・実装
  - **まわす** — GitHub 連携や CI/CD など DevOps フローを構築し、AI を継続的に改善するサイクルを体験
  - **とどける** — Cloud Run へのデプロイで、スケーラブルな環境に本番品質のプロダクトを届ける
- **位置付け**: 単なるプロトタイピングにとどまらず、**実運用を見据えた DevOps プロセス全体**をカバーする
- **最終ピッチ**: 2026/8/19（水）Google 渋谷オフィス（渋谷ストリーム）— 二次審査通過 10 チームを招待

## 5. 審査基準（5 項目）

1. **AI エージェントが価値の中心になっているか**
   - 単なる機能ではなく AI エージェントが作品の中核として機能しているか
   - 自律的な振る舞い（判断・タスク実行）があるか
   - **"AI エージェントである必然性"**があるか
2. **設定した課題へのアプローチ力**
   - 課題と背景、対象ユーザー、提供価値といったプロダクトストーリーの一貫性・妥当性・新規性
3. **ユーザビリティ**
   - 直観的に使える機能やデザイン
4. **実用性・体験価値の魅力**
   - 課題解決への実用性、効果的なアプローチ
   - 提供する体験価値が突き抜けているか（加点）
5. **実装力**
   - 選定した技術・構成の納得度、拡張性、実運用への配慮
   - 必須ツールを活用したうえでの総合的な実現度

## 10. 本プロジェクト（MakePizzaRecipeAgent）への示唆

### 必須で押さえる要素
1. **Google Cloud アプリ実行プロダクト** → **Cloud Run** が最有力（コンセプト「とどける」とも合致）
2. **Google Cloud AI** → **Gemini API**（直接利用可）+ Vertex AI / Agent Builder / ADK の組み合わせ
3. **公開 GitHub リポジトリ** + **動作確認可能なデプロイ URL**
4. **YouTube/Vimeo の紹介動画**
5. **システムアーキテクチャ図**（Proto Pedia 必須）

### 評価軸ごとの戦略

#### ① AI エージェントが価値の中心 / 必然性
- 単発の Gemini API 呼び出しでは弱い → **複数ステップ・ツール使用・自律判断**を含むエージェント設計
- **"なぜエージェントである必要があるのか"** を明確に説明できるストーリーを用意
- 例: 「食材検索 → 組み合わせ評価 → レシピ生成 → 画像生成 → ストーリー生成 → フィードバック取り込み」を Gemini が自律的にオーケストレーション

#### ② 課題へのアプローチ力（ストーリーの一貫性・妥当性・新規性）
- 「ビジネスパートナーとの交流ツールとしてのピザレシピ」という具体的・個人的なユースケースは**新規性で勝負できる**
- 想定ユーザー・課題背景は明確（仙台オフィスでのパートナー交流という実体験）

#### ③ ユーザビリティ
- 直観的な対話 UI（食材入力 → エージェントが対話的に絞り込み）
- モバイル対応（パーティ前のシーンでスマホからも使える）

#### ④ 実用性・体験価値
- **「話のネタになるストーリー生成」**は体験価値で突き抜けるポイントになりうる
- **画像生成（Imagen）**でビジュアル体験を強化
- **フィードバック保存 → 次回提案に活用**は実用性の核

#### ⑤ 実装力（DevOps の実践度）— **最大の差別化要素**
- **CI/CD**: GitHub Actions → Cloud Build / Cloud Deploy → Cloud Run
- **IaC**: Terraform でインフラ定義
- **可観測性**: Cloud Logging / Monitoring / Trace
- **エージェント評価ループ**: フィードバックデータを使った継続的改善（"まわす"の体現）
  - 例: ユーザー評価をもとにプロンプト改善、レシピ品質を自動評価
- これらが**「まわす」コンセプトに直接対応**しており、DevOps 実践度の評価軸を直撃する

### 推奨技術スタック（叩き台）
| レイヤ | 技術 | 理由 |
| --- | --- | --- |
| フロント | Next.js (App Router) | 開発速度、Cloud Run 互換 |
| ホスティング | **Cloud Run** | 必須要件、コンセプト「とどける」に合致 |
| AI Agent | **Gemini API + ADK**（または Agent Builder） | 必須要件、評価軸①の「必然性」を出しやすい |
| 画像生成 | Imagen 3 / Gemini Image | 体験価値、Google Cloud AI の積極活用 |
| DB | Firestore | フィードバック・履歴保存、Google Cloud 統合 |
| IaC | Terraform | DevOps 実践度の証明 |
| CI/CD | GitHub Actions → Cloud Build → Cloud Run | DevOps 実践度の証明 |
| 監視 | Cloud Logging / Monitoring / Trace | DevOps 実践度の証明 |
| エージェント評価 | Vertex AI Gen AI Evaluation Service | "まわす"の体現、独自性 |

### スコープ管理
- 提出締切まで実質 2 ヶ月強。**機能を絞り、DevOps パイプラインを完成させること**を優先
- マルチユーザー化・公開サービス化は v2 として MVP は個人利用想定でも審査上問題なし（むしろ実用性のストーリーが立つ）
